# 仕様メモ

## 目的

選択したフォルダ配下にある対象 Excel ブックを再帰的に探索し、各ブックの先頭シートからテーブル名・カラム名などの一覧を作成する。

## 探索仕様

1. ユーザーがフォルダ選択ダイアログで起点フォルダを選ぶ。
2. 起点フォルダ配下のファイルを再帰的に探索する。
3. Excel の一時ファイル `~$` で始まるファイルは除外する。
4. 拡張子が `.xlsx`、`.xlsm`、`.xlsb`、`.xls` のファイルを候補にする。
5. ファイル名の拡張子を除いたベース名が次のいずれかに完全一致する場合、解析対象にする。
   - `1`
   - `2`
   - `3`
   - `4`
   - `5`
   - `1-` から `5-` の後ろに数字1文字から3文字が続く形式
   - `7-` の後ろに数字1文字から3文字が続く形式

## ファイル名判定案

正規表現で表す場合は次の案を初期候補とする。

```text
^(?:[1-5]|(?:[1-5]|7)-\d{1,3})$
```

ハイフン後の枝番は数字1文字から3文字とし、ゼロ埋めは任意とする。

許可する例:

- `1-1`
- `1-01`
- `1-001`
- `5-999`
- `7-001`

許可しない例:

- `6`
- `6-001`
- `7`
- `7-1000`
- `1-0001`

## 解析仕様

- 対象ブックは読み取り専用で開く。
- 解析対象は先頭シートのみとする。
- テーブル名（英）は先頭シートの `C1` と `B2` から取得候補を探す。
- `C1` と `B2` の両方に値がある場合は、前後空白を除いた文字長が長い方を採用する。
- `C1` と `B2` の両方に値があり、文字長も同じ場合は `C1` を採用する。
- `C1` または `B2` の片方だけに値がある場合は、その値を採用する。
- `C1` と `B2` のどちらからも取得できない場合、テーブル名（英）は `未解決` とする。
- テーブル名（和）とテーブルIDは、対象ブックの先頭シート名から取得する。
- カラム名（英）は、先頭シート内で `フィールド名` と入力されているセルを探し、そのセルの下方向の値から取得する。
- カラム名（和）は、`フィールド名` セルの右隣に `日本語名` と入力されているセルを探し、そのセルの下方向の値から取得する。
- カラム一覧は `フィールド名` 列に値がある行を対象にし、同じ行の `日本語名` 列の値をカラム名（和）として扱う。
- 検索条件は前後空白を除いた完全一致とする。
- 開けないブックや解析中にエラーになったブックはスキップし、処理完了メッセージのスキップ件数に含める。

## 先頭シート名の分解仕様

テーブル名（和）とテーブルIDは、対象ブックの先頭シート名から次のルールで取得する。

1. シート名を文字の塊に分ける。
   - 半角スペース、全角スペース、タブ、括弧、スラッシュ、コロン、セミコロン、カンマ、ハイフンは区切り文字として扱う。
   - `_` は英数字だけの塊の中では区切り文字にしない。ただし、日本語文字に隣接する `_` は区切り文字として扱う。
   - 区切り文字がない場合、英数字または `_` の直後に日本語文字が続く箇所は同じ塊として扱う。
   - 区切り文字がない場合、日本語文字の直後に英数字または `_` が続く箇所は塊を分ける。
2. ひらがな、カタカナ、漢字のいずれかを含む塊は、塊全体をテーブル名（和）候補とする。
3. 日本語を含まず、英字を1文字以上含み、英数字と `_` だけで構成される塊は、テーブルID候補とする。
4. 日本語を含む塊の中にある英字部分は、テーブルID候補として切り出さない。
5. テーブル名（和）候補が複数ある場合は、シート名に出現した順に半角スペースで連結する。
6. テーブルID候補が複数ある場合は、文字長が最も長い候補を採用する。同じ長さの場合は先に出現した候補を採用する。
7. テーブル名（和）またはテーブルIDを取得できない場合は、該当項目を `未解決` とする。

例:

| シート名 | テーブル名（和） | テーブルID |
| --- | --- | --- |
| `EXCEL集計 excels` | `EXCEL集計` | `excels` |
| `顧客マスタ M_CUSTOMER` | `顧客マスタ` | `M_CUSTOMER` |
| `顧客マスタ_M_CUSTOMER` | `顧客マスタ` | `M_CUSTOMER` |
| `EXCEL集計excels` | `EXCEL集計` | `excels` |
| `EXCEL集計` | `EXCEL集計` | `未解決` |
| `excels` | `未解決` | `excels` |

## 出力仕様

実行ブック内に次の4シートを作成する。

### テーブル一覧

対象ブックごとに1行出力する。

| 列名 | 内容 |
| --- | --- |
| テーブル名（英） | 対象ブックの先頭シートの `C1` または `B2`。両方に値がある場合は文字長が長い方、同じ長さの場合は `C1`。未解決の場合は `未解決`。 |
| テーブル名（和） | 対象ブックの先頭シート名から取得する。 |
| テーブルID | 対象ブックの先頭シート名から取得する。 |

### カラム一覧

対象ブックの先頭シートで検出したカラムごとに1行出力する。
可視列は次の3列とし、実装上は検索高速化のため非表示の内部キー列を持つ。

| 列名 | 内容 |
| --- | --- |
| テーブル名（英） | 対象ブックの先頭シートの `C1` または `B2`。両方に値がある場合は文字長が長い方、同じ長さの場合は `C1`。未解決の場合は `未解決`。 |
| カラム名（英） | `フィールド名` セルの下方向にある値。 |
| カラム名（和） | `フィールド名` セル右隣の `日本語名` セルの下方向にある値。 |

### テーブル検索

`テーブル一覧` と同じ列構成にする。
2行目以降をユーザー入力・検索結果領域とし、検索結果はシート上の数式で自動更新する。
初期実装では2行目から1000行目まで数式を設定する。

| 列名 | 内容 |
| --- | --- |
| テーブル名（英） | ユーザーが検索したいテーブル名（英）を入力する。 |
| テーブル名（和） | 入力があり、`テーブル一覧` に一致する行があれば該当値を出力し、一致しない場合は `ERROR` を出力する。入力が空の場合は空欄にする。 |
| テーブルID | 入力があり、`テーブル一覧` に一致する行があれば該当値を出力し、一致しない場合は `ERROR` を出力する。入力が空の場合は空欄にする。 |

### カラム検索

`カラム一覧` と同じ列構成にする。
2行目以降をユーザー入力・検索結果領域とし、検索結果はシート上の数式で自動更新する。
初期実装では2行目から1000行目まで数式を設定する。

| 列名 | 内容 |
| --- | --- |
| テーブル名（英） | ユーザーが検索したいテーブル名（英）を入力する。 |
| カラム名（英） | ユーザーが検索したいカラム名（英）を入力する。 |
| カラム名（和） | 入力があり、`カラム一覧` に一致する行があれば該当値を出力し、一致しない場合は `ERROR` を出力する。入力が空の場合は空欄にする。 |

## Bootstrap 方針

ソース一式をダウンロードできないユーザー向けに、単一ファイルから導入を開始できる仕組みを用意する。

初期実装では `bootstrap/InstallTableAndColumnListMaker.vbs` を Bootstrap ファイルとする。

Bootstrap は GitHub 上の `src/TableAndColumnListMaker.bas` を取得し、デスクトップに `TableAndColumnListMaker.xlsm` を作成して、VBAモジュールを組み込む。
実行には Excel の `VBA プロジェクト オブジェクト モデルへのアクセスを信頼する` 設定が必要になる。
